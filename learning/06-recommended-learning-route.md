# 06. 推荐学习路线

可以按这个节奏来：

1. 读 `README.md` 和 `document/lab1-resolve.md`
2. 看 `TupleDesc`、`Tuple`、`HeapPage`、`HeapFile`
3. 跑对应测试，比如 `TupleTest`、`HeapPageReadTest`
4. 看 `SeqScan`、`Filter`、`Join`
5. 跑执行器相关测试
6. 再进入优化器、事务、索引、日志恢复

最重要的是：不要从 `Parser` 或 `LogicalPlan` 开始。它们在上层，依赖你已经理解底层存储和算子。

建议下一步从 `TupleDesc` 和 `Tuple` 开始：先看它解决什么问题，再看字段和方法，最后用一个小例子串起来。
